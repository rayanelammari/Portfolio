QU'EST-CE QUE LA VEILLE INFORMATIONNELLE ?

La veille informationelle, élément de la veille stratégique, consiste à surveiller les évolutions techniques, les innovations dans un secteur d’activité donnée.

La veille informationelle comprend notamment la surveillance, la collecte, le partage et la diffusion d’information permettant d’anticiper ou de s’informer sur des changements en matière de recherche, développement, brevet, lancement de nouveaux produits, matériaux, processus, concepts, innovation de fabrication, etc….

Cela a pour but d’évaluer l’impact sur l’environnement et l’organisation.



SUJET DE LA VEILLE INFORMATIONNELLE

Pour la veille informationelle, j'ai décidé de traiter sur la vulnérabilité Buffer Overflow.
La thématique est : Comment les attaquants exploitent les vulnérabilités de type Buffer Overflow ?



SUIVI D'INFORMATIONS DE LA VEILLE INFORMATIONNELLE

Les informations arrivent sous forme de mail de google alerte tous les jours des rapports sont envoyés et sont par la suite traités et triés dans la boite mail. Une fois les alertes google triées, les informations utiles sont par la suite ajoutées dans pearltrees, afin de pouvoir les regrouper et pour voir consulter plus facilement les informations importantes. Pour ce qui est de Feedly, je me rends réguliéremenet sur ce site, et lorsqu'une information importante est remontée je mets le lien de l'article dans pearltrees également.




COMMENT EST FAITE LA GESTION DE PEARLTREES ?

Dans Pearltress, j'ai mis en place plusieurs dossiers qui me permet donc de mettre les informations les plus importantes des différentes remontées qui me sont faites. Elles sont tirées, et par la suite je peux me rendre dans le dossier en question afin de voir toutes les informations qui me sont remontées.




QUELQUES DÉFINITIONS

Buffer : un tampon est une zone de stockage temporaire dans la mémoire d'un ordinateur qui contient des données pendant leur traitement. Les tampons sont utilisés dans de nombreuses parties d'un programme, telles que les opérations d'entrée/sortie, la communication réseau et l'allocation de mémoire..

Débordement de tampon/Buffer Overflow : Un débordement de tampon (ou buffer overrun) se produit lorsque le volume de données dépasse la capacité de stockage de la mémoire tampon. En conséquence, le programme qui tente d'écrire les données dans le tampon écrase les emplacements mémoire adjacents .

pointeur en C : Les pointeurs dans les langages bas niveau comme le C/C++ sont des variables spéciales qui stockent les adresses mémoire d'autres variables, permettant ainsi d'accéder et de manipuler directement les données situées à ces adresses.

Buffer overflow basé sur la pile : ce type d'attaque exploite une vulnérabilité dans la pile, qui est une région de mémoire utilisée pour stocker les trames d'appel de fonction et les variables locales. En faisant déborder la pile avec plus de données qu'elle ne peut en contenir, un attaquant peut modifier le comportement du programme et exécuter du code arbitraire.

Buffer overflow basé sur le tas : ce type d'attaque cible le tas, qui est une région de mémoire utilisée pour l'allocation dynamique de mémoire. En faisant déborder le tas avec plus de données qu'il ne peut en contenir, un attaquant peut corrompre l'espace mémoire et modifier le comportement du programme.
